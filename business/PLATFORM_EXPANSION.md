# OASIS 多平台扩展方案

## 一、扩展一个新平台需要改什么

OASIS 当前支持 Twitter 和 Reddit。扩展到任何新平台，本质上只需要改三样东西：

```
┌────────────────────────────────────────────────┐
│            扩展一个新平台 = 改三层              │
│                                                │
│  1. 动作空间（ActionType）                      │
│     → 这个平台的用户能做什么                     │
│                                                │
│  2. 推荐算法（RecSys）                          │
│     → 这个平台怎么给用户推内容                   │
│                                                │
│  3. 用户画像 Prompt（UserInfo）                  │
│     → Agent 以什么身份、什么语气行动              │
│                                                │
│  数据库结构、Channel 通信、环境编排 → 基本不用改  │
└────────────────────────────────────────────────┘
```

### 2.2 优先级排序

```
第一梯队（先做）：小红书 > 微博
  理由：与现有 Twitter/Reddit 模型最接近，改造量最小，客户需求最明确

第二梯队（跟进）：抖音 > 微信视频号
  理由：市场最大，但需要引入"视频内容描述"机制，改造量中等

第三梯队（后做）：B站 > 快手
  理由：场景独特，需要更多定制
```

## 三、各平台详细扩展方案

### 3.1 小红书

**与现有 Reddit 模式最接近**，改造量最小。

#### 动作空间

| 小红书动作 | 映射到现有 ActionType | 需要新增？ |
|-----------|---------------------|-----------|
| 发布笔记 | `CREATE_POST` | 不需要，content 改为标题+正文 |
| 点赞 | `LIKE_POST` | 不需要 |
| 收藏 | — | **需要新增** `COLLECT_POST` |
| 评论 | `CREATE_COMMENT` | 不需要 |
| 关注 | `FOLLOW` | 不需要 |
| 搜索 | `SEARCH_POSTS` | 不需要 |
| 分享 | — | **可选新增** `SHARE_POST` |
| 举报 | `REPORT_POST` | 不需要 |

新增动作仅 1-2 个，工作量约 1 天。

#### 推荐算法

小红书的推荐是 **搜索+推荐双列流**：
```
用户打开小红书
├── 发现页（推荐流）→ 类似 Reddit 热度排序 + 兴趣匹配
│   可复用现有 rec_sys_reddit + rec_sys_personalized_twh 的混合
│
└── 搜索页 → 用户主动搜索关键词
    可复用现有 SEARCH_POSTS 功能
```

关键差异：小红书的推荐重度依赖 **"收藏"和"搜索"** 行为，而非单纯的 like。
改造：在热度分计算中加入收藏权重（收藏权重 > 点赞权重）。

### 3.2 微博

**与现有 Twitter 模式几乎一致**，改造量最小。

#### 动作空间

| 微博动作 | 映射到现有 ActionType | 需要新增？ |
|---------|---------------------|-----------|
| 发微博 | `CREATE_POST` | 不需要 |
| 点赞 | `LIKE_POST` | 不需要 |
| 转发 | `REPOST` | 不需要 |
| 评论 | `CREATE_COMMENT` | 不需要 |
| 关注 | `FOLLOW` | 不需要 |
| 搜索 | `SEARCH_POSTS` | 不需要 |
| 看热搜 | `TREND` | 不需要 |
| 引用转发 | `QUOTE_POST` | 不需要 |

**零新增**，只需要改 Prompt 语言和推荐算法权重。

#### 推荐算法

微博的核心分发是 **热搜机制**：
```
微博信息流
├── 关注流 → 直接复用现有 following_post 逻辑
├── 推荐流 → 复用现有 rec_sys_personalized_twh
└── 热搜榜 → 复用现有 TREND 功能，调整热度计算公式
    热搜分 = 搜索量 × 讨论量 × 时间衰减
    改造：在现有 hot_score 基础上加入"讨论量"维度
```

#### 商业场景价值

- **热点营销预判**：模拟某话题能否上热搜、讨论走向
- **PR 危机推演**：模拟负面事件在微博的发酵过程
- **大V 投放效果**：模拟不同大V 转发后的二次传播链路

---

### 3.3 抖音

**需要引入"视频内容"的文本化描述机制**，改造量中等。

#### 核心差异

抖音的内容是短视频，但 OASIS 中的 Agent 是文本驱动的。解决方案：

```
视频内容 → 文本化描述
│
├── 方案 A：用结构化描述替代视频
│   {
│     "title": "3 个拉伸动作改善体态",
│     "description": "健身博主示范3个简单拉伸动作...",
│     "tags": ["健身", "拉伸", "体态矫正"],
│     "style": "教程类",
│     "duration": "45秒",
│     "bgm": "励志音乐"
│   }
│
└── 方案 B：把视频 prompt 当作内容
    Agent 看到的是视频描述文本，基于文本做决策
    这与真实用户"刷到封面→决定是否看→看完决定互动"的过程一致
```

#### 动作空间

| 抖音动作 | 映射到现有 ActionType | 需要新增？ |
|---------|---------------------|-----------|
| 发布视频 | `CREATE_POST`（content 为视频描述） | 不需要 |
| 双击/点赞 | `LIKE_POST` | 不需要 |
| 评论 | `CREATE_COMMENT` | 不需要 |
| 转发 | `REPOST` | 不需要 |
| 关注 | `FOLLOW` | 不需要 |
| 不感兴趣 | `DISLIKE_POST` | 不需要 |
| 滑走（不互动） | `DO_NOTHING` | 不需要 |
| 收藏 | — | **需要新增** |
| 合拍/拼接 | — | **可选新增**（低优先） |
| 购物车点击 | `PURCHASE_PRODUCT` | 不需要 |

#### 推荐算法（关键差异）

抖音是 **纯算法推荐**，几乎不依赖社交关系：

```python
    3. 互动率加权（点赞/评论/转发/收藏 的权重不同）
    4. 流量池递进（初始 300-500 曝光 → 数据好则进入下一级流量池）
    5. 时间衰减（新内容优先）
    """

```

抖音推荐的独特之处是 **流量池机制**，可以通过在 `rec` 表中分阶段更新来模拟：
```
第 1 轮 step：新视频只推给 50 个 Agent（初始流量池）
第 2 轮 step：如果互动率 > 阈值，推给 200 个 Agent（二级流量池）
第 3 轮 step：如果继续超标，推给全部 Agent（爆款）
```

#### 商业场景价值

- **短视频内容测试**：不同风格/话题/标题的完播率和互动率预判
- **DOU+ 投放模拟**：模拟加热后的流量分配和 ROI
- **直播预热策略**：模拟不同预热内容对直播间引流的效果
- **带货转化预测**：模拟商品链接点击率和购买意向

### 3.4 快手

#### 核心差异：强社交 + 下沉市场

快手的推荐更依赖社交关系（"老铁"文化），与抖音的纯算法推荐形成对比。

```
快手推荐 = 社交推荐（60%）+ 算法推荐（40%）
├── 关注页权重远高于抖音
├── 同城内容占比更大
└── "双列展示"让用户有主动选择权（vs 抖音单列强推）
```

#### 动作空间新增

| 快手特有动作 | 说明 |
|-------------|------|
| 送礼物 | 直播场景核心互动 |
| 说说（类朋友圈） | 图文动态 |

#### 商业场景价值

- **下沉市场策略验证**：模拟三四线城市用户对品牌的认知和接受度
- **直播电商推演**：模拟直播间互动节奏和转化漏斗
- **老铁裂变传播**：模拟社交关系链驱动的内容传播

### 3.5 B站

#### 核心差异：弹幕文化 + 社区氛围

| B站特有要素 | OASIS 扩展方式 |
|------------|---------------|
| 弹幕 | 新增 `SEND_DANMAKU` 动作，类似评论但更短更即时 |
| 投币 | 新增 `GIVE_COIN` 动作（用户每天限额） |
| 三连（点赞+投币+收藏） | 组合动作，LLM 可以一次触发三个 tool call |
| 长视频 | content 描述更长，包含分 P 信息 |
| 专栏文章 | 复用 `CREATE_POST` |

#### 商业场景价值

- **品牌年轻化内容测试**：模拟 Z 世代对品牌内容的反应
- **UP 主合作效果预判**：模拟不同 UP 主的粉丝画像与品牌的匹配度
- **弹幕舆情分析**：模拟视频发布后弹幕中的关键词和情绪分布

### 3.6 微信视频号

#### 核心差异：社交裂变 + 私域联动

这是改造量最大的平台，因为涉及 **私域社交关系链**（朋友圈、微信群、公众号）。

```
视频号分发 = 社交推荐（朋友点赞可见） + 算法推荐
├── 好友点赞 → 出现在朋友的"朋友点赞"tab 中
├── 微信群分享 → 可复用现有 GROUP 功能
└── 公众号关联 → 需要新增"文章→视频"引流逻辑
```

#### 商业场景价值

- **私域裂变模拟**：一条视频通过微信好友链的传播路径
- **本地生活推广**：模拟同城用户的内容分发效果

## 五、多平台带来的新商业场景

### 5.1 跨平台投放策略优化

这是单一平台仿真做不到的，也是最大的差异化卖点：

> "同一个产品，同样 10 万预算，
> 分别在小红书、抖音、微博投放，
> AI 仿真告诉你最优的预算分配比例。"

```
实验设计：
├── 产品：某国货护肤品
├── 预算：10 万
├── 策略 A：小红书 5 万 + 抖音 3 万 + 微博 2 万
├── 策略 B：抖音 6 万 + 小红书 3 万 + 微博 1 万
├── 策略 C：小红书 8 万 + 微博 2 万（不投抖音）
│
└── 仿真指标对比：
    ├── 曝光量
    ├── 互动率（点赞/评论/收藏/转发）
    ├── 种草指数（收藏率 + 正面评论比例）
    ├── 讨论深度（评论中提到产品功能的比例）
    └── 跨平台溢出（在 A 平台看到后去 B 平台搜索的比例）
```

**这个场景可以把客单价从单平台的 5000-8000 元提升到 2-5 万元。**

### 5.2 平台特性对比研究

帮品牌方理解不同平台的用户行为差异：

> "同一条内容发在小红书和抖音上，用户反应有什么不同？"

- 小红书用户更关注**成分和细节**，评论倾向于讨论功效
- 抖音用户更关注**视觉冲击和情绪共鸣**，评论倾向于表达感受
- 微博用户更关注**话题性和争议性**，评论倾向于站队和对比

### 5.3 全域营销推演

```
用户旅程仿真：
  微博热搜看到品牌 → 小红书搜索种草笔记 → 抖音刷到广告 → 下单购买

每个环节的流失率、转化率、关键影响因素都可以通过多平台仿真来量化。
```

## 六、与商业化方案的衔接

### 定价体系扩展

| 服务 | 价格 | 说明 |
|------|------|------|
| 单平台仿真 | 5000-8000 元/次 | 小红书/微博/抖音任选一个 |
| 双平台对比 | 1.2-1.8 万元/次 | 两个平台相同内容对比 |
| 三平台全域 | 2-3 万元/次 | 三个平台 + 跨平台分析 |
| 全域营销推演 | 3-5 万元/次 | 全平台 + 用户旅程 + 预算分配建议 |

### 获客话术升级

单平台：
> "帮你在投放前预演小红书种草效果"

多平台（溢价更高）：
> "同样的预算，告诉你在小红书、抖音、微博之间怎么分配最划算"

## 七、总结：扩展顺序与节奏

```
第 1 个月：小红书 + 微博（改造量最小，验证市场）
     │
第 2-3 个月：用这两个平台跑通商业模式、积累案例
     │
第 4-5 个月：加入抖音（市场最大，需要流量池机制）
     │
第 6 个月后：根据客户需求决定 快手/B站/视频号 的优先级
     │
长期：打造"全域社交仿真平台"的定位
```

**核心策略：不要一次做六个平台。先做最容易的两个验证商业模式，再根据客户需求逐步扩展。每多支持一个平台，就多一个涨价的理由。**
