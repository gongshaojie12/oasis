# OASIS 低成本启动与变现计划

## 核心原则

**不做平台，先做项目；不建团队，先跑通单子；不追完美，先证明价值。**

---

## 一、冷启动成本估算

### 最低启动配置（1-2 人团队）

| 项目 | 月成本 | 说明 |
|------|--------|------|
| LLM API | 500-2000 元 | 用 DeepSeek/Qwen API，比 GPT-4o 便宜 10-20 倍 |
| 云服务器 | 200-500 元 | 轻量云主机跑 Demo，不需要 GPU |
| 域名+网站 | 50 元 | 简单落地页即可 |
| 办公成本 | 0 | 远程办公 |
| **月合计** | **约 1000-3000 元** | |

### 可选追加

| 项目 | 成本 | 触发条件 |
|------|------|---------|
| GPU 服务器 | 2000-5000 元/月 | 客户需要大规模仿真（>1万 Agent）|
| vLLM 本地部署 | 一次性 5000-15000 元 | 量大后降成本、或客户要求数据不出境 |
| 设计/前端外包 | 3000-8000 元 | 做正式的产品 Demo 页面 |

**结论：3000 元/月即可启动，首笔收入前总投入控制在 2-3 万以内。**

---

## 二、最快变现路径：咨询+工具 模式

不要一上来就做 SaaS 平台。做平台要烧钱、要团队、要时间。正确的起步方式是：

```
阶段一：卖服务（咨询项目制）    →  验证需求、积累案例、产生现金流
阶段二：卖工具（标准化产品）    →  复用项目经验、提升利润率
阶段三：卖平台（SaaS 订阅制）  →  有了客户基础再做
```

## 三、具体执行计划

### 第一阶段：验证期（第 1-2 个月）

#### 目标：签下第 1 个付费客户

**做什么**：

1. **选定 1 个垂直场景，做出 1 个震撼 Demo**

   推荐场景：**小红书/微博种草营销仿真**（国内市场最容易变现）

   Demo 内容：
   - 模拟 500 个不同画像的用户（年龄、性别、兴趣标签）
   - 投放一条美妆/3C 产品的种草笔记
   - 展示 24 小时内的传播链路：谁点赞了、谁评论了、谁转发了、讨论走向如何
   - 对比两种投放策略（纯素人 vs KOL+素人）的效果差异

   技术改造（基于现有 OASIS 代码）：
   ```
   改造量很小，核心是：
   ├── 把 Agent 画像改成中文用户画像（改 UserInfo 的 system prompt）
   ├── 把 ActionType 的描述改为小红书/微博的语境
   ├── 写一个简单的结果分析脚本（从 SQLite trace 表提取数据）
   └── 用 Streamlit 做一个可视化页面（1-2 天能搞定）
   ```

2. **准备销售材料**

   - 1 页纸方案（客户能在 30 秒内看懂你能帮他做什么）
   - 1 个 3 分钟的 Demo 录屏视频
   - 1 个案例报告（用 Demo 数据生成的仿真分析报告）

3. **找到第一批潜在客户**

   精准渠道（不要广撒网）：
   | 渠道 | 方式 | 目标 |
   |------|------|------|
   | 小红书/知乎 | 发技术文章"我用 AI 模拟了 500 个小红书用户" | 吸引品牌方/MCN 主动联系 |
   | 即刻/Twitter | 技术圈传播 | 吸引早期采用者 |
   | 微信社群 | 品牌营销/新消费社群 | 直接触达决策者 |
   | 朋友圈 | 熟人推荐 | 转介绍是最高效的获客方式 |

   **关键话术**：
   > "你的下一次营销投放，可以先用 AI 模拟 500 个真实用户的反应，
   > 提前知道哪种内容、哪种 KOL 组合效果最好，避免浪费预算。"

4. **定价策略**

   第一单不要贪多，目标是**成交**：
   | 服务包 | 价格 | 内容 |
   |--------|------|------|
   | 体验版 | 免费 | 50 Agent 的迷你仿真 Demo |
   | 基础版 | 5000-8000 元/次 | 500 Agent、单次营销仿真 + 报告 |
   | 专业版 | 1.5-3 万元/次 | 2000 Agent、多策略对比 + 深度分析报告 |

   **给第一个客户打五折甚至免费，换取案例授权和推荐信。**

### 第二阶段：跑通期（第 3-4 个月）

#### 目标：累计 3-5 个付费客户，月收入 1-3 万

**做什么**：

1. **用第一个案例撬动更多客户**

   - 把第一个客户的案例（脱敏后）做成标准案例
   - 在行业社群和内容平台持续分享
   - 争取客户口碑推荐

2. **标准化交付流程**（降低每单边际成本）

   ```
   客户下单
   │
   ├─ 1. 需求沟通（30 分钟电话）→ 明确：产品类型、目标人群、投放策略
   ├─ 2. 画像配置（1 小时）→ 根据客户目标人群配置 Agent 画像
   ├─ 3. 仿真运行（2-4 小时）→ 运行仿真、收集数据
   ├─ 4. 报告生成（2-3 小时）→ 半自动化报告生成
   └─ 5. 交付沟通（30 分钟）→ 讲解报告、收集反馈

   单次交付总耗时：约 1 个工作日
   ```

3. **拓展第二个场景：舆情推演**

   用同样的技术底座，换一套话术卖给不同客户：
   | 客户类型 | 场景 | 话术 |
   |---------|------|------|
   | 上市公司 PR 部门 | 财报发布前舆论预判 | "提前 48 小时知道市场怎么讨论你的财报" |
   | 政府宣传部门 | 政策发布舆论推演 | "政策发布前模拟公众反应，准备好应对预案" |
   | 大型企业品牌部 | 危机公关推演 | "危机来临前演练 10 种应对方案" |

   **定价更高**：单次 2-5 万，因为决策价值大。

4. **技术优化**（只做必要的）

   | 优化项 | 目的 | 耗时 |
   |--------|------|------|
   | 报告模板自动生成 | 减少每单人工时间 | 3-5 天 |
   | Agent 画像模板库 | 快速配置常见用户群体 | 2-3 天 |
   | 接入国产低成本 LLM | 降低 API 成本到 1/10 | 1-2 天 |
   | Streamlit Dashboard 优化 | 客户自助查看结果 | 3-5 天 |

### 第三阶段：增长期（第 5-8 个月）

#### 目标：月收入 5-10 万，开始做产品化

**做什么**：

1. **推出月度订阅服务**

   对于有持续需求的客户（MCN、品牌方），推出订阅制：

   | 套餐 | 月费 | 内容 |
   |------|------|------|
   | 基础版 | 8000 元/月 | 每月 4 次仿真、500 Agent |
   | 专业版 | 2 万元/月 | 每月 8 次仿真、2000 Agent、定制画像 |
   | 企业版 | 5 万元/月 | 不限次数、5000 Agent、专属支持 |

2. **做一个简单的自助平台**

   不需要复杂的全功能 SaaS，做一个最小化的自助界面：
   ```
   ├── 选择仿真场景（营销/舆情/产品）
   ├── 上传或选择 Agent 画像模板
   ├── 输入要测试的内容/策略
   ├── 一键运行仿真
   └── 在线查看报告 + PDF 导出
   ```
   技术选型：Streamlit / Gradio → 最快 1-2 周上线

3. **建立内容获客飞轮**

   ```
   发布仿真案例文章
       ↓
   吸引潜在客户咨询
       ↓
   提供免费 Demo 体验
       ↓
   转化为付费客户
       ↓
   产出新案例
       ↓
   (循环)
   ```

   高价值内容选题：
   - "我用 AI 模拟了 1000 个用户，预测了 XX 品牌的营销效果"
   - "同样的预算，哪种 KOL 投放组合 ROI 最高？AI 仿真给出答案"
   - "政策发布前的舆论推演：AI 如何帮助决策者预判公众反应"

4. **考虑招第一个人**

   当月收入稳定在 5 万以上时，考虑招 1 个人：
   - 优先招一个懂营销/运营的人，负责获客和客户维护
   - 技术开发仍由创始人自己做

---

### 第四阶段：规模化（第 9-12 个月）

#### 目标：月收入 10-20 万，形成可复制的商业模式

**做什么**：

1. **打磨标准化产品**，降低交付成本
2. **拓展代理/合作渠道**：找 1-2 家营销服务商做渠道合作
3. **沉淀行业数据**：积累不同行业的仿真数据和 Agent 画像库，形成数据壁垒
4. **考虑融资或保持自盈利**：根据增长情况决定

## 四、成本控制关键策略

### 4.1 LLM 成本控制（最大的变动成本）

```
策略一：分层模型
├── 核心 Agent（KOL、意见领袖）→ GPT-4o / Claude → 高质量决策
├── 普通 Agent（大众用户）→ DeepSeek V3 / Qwen → 成本极低
└── 背景 Agent（活跃度低的用户）→ 规则引擎，不调用 LLM

策略二：缓存复用
├── 相同画像 + 相似场景的 Agent 响应可缓存
└── 减少重复 LLM 调用

策略三：国产模型优先
├── DeepSeek V3：约 1 元/百万 token（GPT-4o 的 1/100）
├── Qwen-Plus：约 4 元/百万 token
└── 通义千问开源版 + vLLM 本地部署：趋近于 0 成本
```

**成本对比示例（500 Agent 单次仿真）**：

| 模型方案 | 预估 Token | API 成本 |
|---------|-----------|---------|
| 全部用 GPT-4o | ~500 万 token | ~100-150 元 |
| 全部用 DeepSeek V3 | ~500 万 token | ~5-10 元 |
| 分层（50 核心 + 450 普通） | ~500 万 token | ~15-25 元 |

**结论：用国产模型，单次仿真成本可控制在 10-25 元，客户付费 5000+，毛利率 >95%。**

### 4.2 技术成本控制

| 不要做 | 替代方案 |
|--------|---------|
| 自建前端 React/Vue 应用 | 用 Streamlit / Gradio，1-2 天出页面 |
| 自建用户系统/支付系统 | 用微信/支付宝收款码，人工管理客户 |
| 自建 GPU 集群 | 按需用云 GPU（AutoDL 约 1-3 元/小时）|
| 自建部署运维体系 | 直接跑在单台云主机上 |
| 完善所有功能再上线 | 先用 Jupyter Notebook 跑出结果再说 |

### 4.3 时间成本控制

**1-2 人团队的时间分配**：

| 活动 | 占比 | 说明 |
|------|------|------|
| 获客+销售 | 40% | 前期最重要的事 |
| 交付项目 | 30% | 产生收入 |
| 产品优化 | 20% | 只做对获客和交付有直接帮助的 |
| 其他 | 10% | 学习、社区维护等 |

**关键认知：技术创始人最大的陷阱是把 80% 的时间花在打磨产品上，但真正该做的是花 40% 以上的时间在获客上。**

## 六、关键里程碑与判断标准

| 时间点 | 里程碑 | 达到则继续 | 未达到则调整 |
|--------|--------|-----------|------------|
| 第 1 个月末 | 有 3+ 个客户表示感兴趣 | 继续推进 | 换场景/换话术/换渠道 |
| 第 2 个月末 | 签下第 1 个付费客户 | 加大获客力度 | 深入调研客户真实痛点 |
| 第 4 个月末 | 累计收入 3 万+ | 开始做标准化 | 考虑是否 Pivot 到其他方向 |
| 第 6 个月末 | 月收入稳定 3-5 万 | 开始招人、做产品 | 审视商业模式是否成立 |
| 第 12 个月末 | 月收入 10 万+ | 考虑融资或持续自盈利 | 决定是继续还是转型 |

## 七、风险预案

| 风险 | 信号 | 应对 |
|------|------|------|
| 客户觉得"仿真不够真" | Demo 阶段客户质疑准确度 | 不卖"预测"，卖"洞察"和"预演"；强调"比拍脑袋决策好" |
| 找不到付费客户 | 1 个月内 0 付费意向 | 降价到 2000 元/次试水；换目标客户群体 |
| 技术问题影响交付 | 仿真结果不稳定 | 减少 Agent 数量、用更稳定的模型、增加规则兜底 |
| LLM 成本超预期 | 单次成本 > 100 元 | 减少 Agent 数、增加规则 Agent 比例、换更便宜的模型 |
| 被客户要求过度定制 | 每个客户都要不同功能 | 只接标准化程度高的需求，拒绝重度定制 |
